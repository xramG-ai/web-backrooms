# WEB BACKROOMS

브라우저에서 실행되는 3D 백룸 탐색 프로젝트.

노란 벽지의 공간을 무한히 걷는다. 규칙은 문서화되어 있지 않다.

**플레이:** https://xramg-ai.github.io/web-backrooms/

## 진입

타이틀 화면의 금속 크립텍스 장치가 입구다.

- 세로 링 10개, 각 링은 `-` + A~Z (27자)
- 휠 또는 ▲▼ 버튼으로 링을 돌려 열쇠말을 맞추면 잠금이 풀린다
- 시작 시 10자 중 7자는 맞춰져 있고, 무작위 3자리만 어긋나 있다
- 열쇠말은 어디에도 적혀 있지 않다

## 컨셉 — 초기 기획

이 프로젝트는 세 가지 감각을 결합한 기획에서 출발했다.

1. **백룸** — 노란 벽지, 반복되는 실내, 낮은 가시거리. 익숙하지만 설명할 수 없는 공간.
2. **영화 「큐브」의 공간 원리** — 전체 구조를 외우거나 지도화할 수 없다. 사용자가 이동하는 동안
   통로와 방의 연결이 바뀌고, 
   그 변화는 사용자가 볼 수 없는 순간에 일어난다.
   공간은 넓어서가 아니라 연결이 계속 달라지기 때문에 무한하게 느껴진다.
3. **글리치** — 벽 관통, 바닥 관통 같은 현상을 버그가 아니라 세계관 요소로 쓴다.
   허용 범위를 벗어난 이동은 시스템이 "글리치"로 판정하고, 이후 더 가혹한 공간 규칙이 적용된다.

핵심 구조:

- 전체 백룸은 **100 × 100 × 100 논리 좌표**를 가진 시스템이다. 세로축은 층이다.
- 사용자는 수평 중앙 부근, 중간 층대에서 시작하지만 그 사실을 알 수 없다.
- 정상 층 범위가 있고, 위나 아래로 벗어나면 이탈 구역 — 공간이 점점 잘못 작동하기 시작한다.
- 유효 시야는 정면 약 10m. 안개·조명 감쇠·심도 블러로 구현한다.
- 정상적으로 걷다 보면 언젠가 문이 나타난다. 문은 출구가 아니며, 아무것도 보장하지 않는다.
- 조작 누적에 따라 엔티티의 얼굴이 사진 플래시처럼 순간 나타날 수 있다.
- 사용자에게는 좌표·층수·규칙을 절대 알려주지 않는다. 시스템은 모든 것을 알고 있다.

> 공간은 존재하지만 이해할 수 없고, 길은 이어지지만 어디로 가는지 알 수 없으며,
> 시스템은 사용자를 보고 있지만 사용자는 시스템을 볼 수 없다.

상세 기획: [docs/PLAN.md](docs/PLAN.md)

## 조작

| 동작 | 키 |
|---|---|
| 이동 | W A S D |
| 달리기 | SHIFT |
| 점프 | SPACE |
| 앉기 | CTRL 또는 C |
| 줍기 | F (아이템 시스템 예정) |
| 소지품 | TAB |
| 일시 정지 | ESC |

게임은 전체화면 + 포인터 락 + Keyboard Lock으로 실행된다 (Chrome/Edge에서는 Ctrl+W도 게임 입력으로 흡수).

ESC 일시 정지 메뉴: 계속하기 / 설정 / 메뉴로 나가기. 설정에서 시야각(70~110), 마우스 감도, 거리 블러, 볼륨 조절. 메뉴로 나가면 크립텍스는 다시 잠긴다.

## 바로 실행 (로컬)

`dist/index.html`을 브라우저에서 열면 끝. (단일 파일 빌드 — 서버 불필요)

## 구현 현황

### M0 — 걷는 백룸 ✅
- 1인칭 컨트롤러 (포인터 락, 헤드밥, 발소리 연동, 앉기)
- 시드 기반 결정론적 청크 생성 (20m 청크) — 같은 곳으로 돌아가면 같은 구조
- 청크 경계에 벽 없음 → 전역 연결 보장 (갇힘 방지)
- AABB 슬라이드 충돌 (M3에서 Rapier + 균열 설계로 교체 예정)
- 지수 안개 + 심도 블러(DoF)로 유효 시야 ~10m 제한
- 절차 생성 텍스처 (노란 벽지·카펫·천장 타일 — 외부 이미지 자산 0)
- 절차 생성 오디오 (형광등 험, 카펫 발소리, 크립텍스 클릭/해제음 — 오디오 파일 0)
- 벽 UV 월드 크기 매핑, 청크당 병합 지오메트리 (드로우콜 최소화)

### UI / UX ✅
- 크립텍스 자물쇠 타이틀 화면 (금속판, 리벳, 각인, 회전 링)
- 잠금 해제 시 "진입" 버튼 (휠 조작은 브라우저 제스처로 인정되지 않는 문제 대응)
- 금속판 일시 정지 메뉴 + 설정 (ESC 뒤로가기 지원)
- Tab 소지품 UI (8칸, 커서 표시)
- 원형 매립 조명, 벽과 겹치는 조명 자동 제외

### 성능 ✅
- 청크 로드/해제 히스테리시스 + 프레임당 최대 2개 마운트 (경계 히치 제거)
- 렌더 픽셀 비율 1.5 상한 (고DPI 프레임 확보)

### M1 — 플로팅 오리진 ✅
- 청크 지오메트리 로컬 좌표 베이크 + 월드 오프셋 그룹
- 기준점에서 240m 이탈 시 리베이스 (같은 프레임에 카메라·월드 동시 이동 → 이음새 없음)
- 논리 좌표는 배정밀도로 무한 확장, GPU에는 항상 작은 좌표만 전달
- 검증: 원점에서 1,000km 지점 텔레포트 후 렌더 좌표 (10,10) 유지 확인
- 누적 보행 거리 추적 (`?debug`로 상태 확인 가능)

## 개발 이력 — 플레이 피드백으로 바뀐 것들

첫 빌드 이후 실제 플레이 피드백을 받아 순서대로 반영했다.

1. **앉기 추가** — Ctrl(또는 C)로 앉기. 이동 감속, 점프 제한.
2. **조명 전체 점등** — 초기 플리커/소등 연출이 산만하다는 피드백으로 정상 구역은 전부 켜짐.
   플리커는 이후 이탈 구역 전용 연출로 되살릴 예정.
3. **원거리 블러** — 10m 너머는 안개만이 아니라 심도 블러(DoF)로 흐려지게.
4. **탭이 꺼지는 문제** — 앉기(Ctrl)+이동(W)이 브라우저의 Ctrl+W(탭 닫기)로 먹히던 것.
   전체화면 + Keyboard Lock API로 게임이 단축키를 흡수하도록 수정.
5. **Tab 소지품 UI** — 인벤토리 프레임 추가. 열 때 포인터 락을 풀어 커서가 보이게.
6. **ESC → 시작화면이 아니라 메뉴** — UI 상태 머신 도입 (타이틀/게임/소지품/일시정지).
7. **타이틀 개명** — "노란 방" → **WEB BACKROOMS**.
8. **크립텍스 진입 장치** — 장식이 아니라 실제로 돌려서 여는 자물쇠로.
   세로 링 10개(- + A~Z), 열쇠말을 맞추면 진입. 접근성을 위해 7자는 미리 맞춰져 있게.
   힌트는 의도적으로 제공하지 않음.
9. **정답을 맞춰도 진입이 안 되던 버그** — 휠 회전은 브라우저가 사용자 제스처로
   인정하지 않아 전체화면 요청이 거부되던 것. 잠금 해제 시 "진입" 버튼으로 해결.
10. **일시 정지 메뉴판** — 계속하기 / 설정 / 메뉴로 나가기. 나가면 자물쇠는 다시 잠긴다.
11. **시야각 확대 + 설정 슬라이더**, **설정에서 ESC 뒤로가기**, **이동 끊김 개선**
    (청크 생성 분산 + 픽셀 비율 상한).
12. **벽-조명 관통 수정** — 벽이 지나가는 자리에는 조명을 설치하지 않음.
    조명은 사각 패널에서 **원형 매립등**으로. **F키(줍기)** 예약.

### 로드맵
- **M2** 논리 좌표(100×100×100, 층 개념) + 시야 밖 공간 재배치 (에포크 기반)
- **M3** 글리치 판정 + 이탈 구역 (의도된 균열, Rapier 도입)
- **M4** 문 스케줄러 (1시간 규칙) + 엔티티 얼굴 플래시 + 구역별 오디오

## 구조

```
src/
  game/
    config.ts     상수 (속도, 시야, 청크, 리베이스 거리, 시드)
    rng.ts        해시 + mulberry32
    chunks.ts     청크 생성 규칙 + 캐시
    collision.ts  AABB 수집 + 축 분리 슬라이드
    textures.ts   캔버스 절차 텍스처 + 재질
    audio.ts      Web Audio 사운드 (험/발소리/크립텍스)
    state.ts      프레임 전역 상태 (React 밖)
  components/
    World.tsx     청크 로드/언로드 + 플로팅 오리진 그룹
    ChunkMesh.tsx 청크 지오메트리 병합 (로컬 좌표) + 조명
    Player.tsx    이동/시점/중력/충돌/리베이스/헤드밥
    Effects.tsx   심도 블러 포스트프로세싱
  App.tsx         UI 상태 머신 (크립텍스/게임/소지품/일시정지)
docs/PLAN.md      기획서 v2
```
